## 概览
思维导图使用 Markdown 格式描述，通过缩进层级表示节点关系。markmap 会将 Markdown 渲染为可视化的思维导图。

## Markdown 格式

### 基本结构
```markdown
# 中心主题

## 一级节点
### 二级节点
#### 三级节点

## 另一个一级节点
### 子节点
```

## 基础语法

### 层级表示
- 使用 `#` 表示节点层级
- `#` 后加空格，然后是节点文本
- 层级关系：`#` (中心) → `##` (一级) → `###` (二级) → `####` (三级)

### 示例
```markdown
# 项目管理

## 需求阶段
### 需求调研
### 需求分析
### 需求评审

## 设计阶段
### 架构设计
### 接口设计
### 数据库设计

## 开发阶段
### 前端开发
### 后端开发
### 联调测试
```

## 高级特性

### 代码块
可以在节点中添加代码块：
```markdown
# 技术栈

## 前端
### 框架
```
Vue.js
React
Angular
```
```

### 链接
```markdown
# 学习资源

## 官方文档
[Vue.js 官方文档](https://vuejs.org)

## 教程
[Vue 教程](https://tutorial.com)
```

### 图标
markmap 支持 emoji 和图标：
```markdown
# 项目计划

## 阶段1 📋
需求调研
需求分析

## 阶段2 🎨
UI 设计
交互设计

## 阶段3 💻
前端开发
后端开发
```

### 标签和注释
```markdown
# 功能模块

## 用户模块 <!-- 重要 -->
登录
注册
个人中心

## 订单模块 <!-- 优先级高 -->
下单
支付
退款
```

## 最佳实践

### 节点命名
- 使用简洁、清晰的语言
- 同一层级的节点命名风格一致
- 避免过长的文本

### 层级结构
- 控制层级深度（建议不超过 5 层）
- 同一层级的节点数量适中（建议 3-7 个）
- 逻辑关系清晰，避免交叉引用

### 内容组织
- 按时间顺序、重要程度或逻辑关系组织
- 使用 emoji 增强可视化效果
- 必要时添加注释说明

## 输出格式
- SVG：矢量图，可无损缩放，推荐使用
- PNG：位图，需要额外工具转换

## 注意事项
- 必须以 `#` 开始中心主题
- 每个节点独占一行
- 使用空格缩进或 `#` 数量表示层级
- 支持中文，无需特殊处理
- markmap 默认生成 SVG 格式
