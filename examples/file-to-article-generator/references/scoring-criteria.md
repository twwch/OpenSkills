## 评分维度

对生成的文章从以下5个维度进行评分：

| 维度 | 权重 | 说明 |
|-----|-----|------|
| 内容准确性 | 35% | 是否准确反映原文核心信息，无严重遗漏或歪曲 |
| 结构完整性 | 20% | 文章结构是否清晰、逻辑是否连贯、层次是否分明 |
| 语言质量 | 15% | 语言表达是否流畅、用词是否准确、风格是否合适 |
| 配图契合度 | 15% | 图片是否与文章内容相关，配图位置是否恰当，图片与文字是否形成良好配合 |
| 需求匹配度 | 15% | 是否满足用户的特定需求（类型、风格、重点等） |

每个维度满分100分，总分按加权计算。

## 评分细则

### 1. 内容准确性（40分）

| 评分 | 标准 |
|-----|------|
| 90-100分 | 完整准确反映原文所有核心信息，无遗漏、无歪曲，信息层次清晰 |
| 80-89分 | 准确反映原文主要信息，遗漏次要细节但不影响理解 |
| 70-79分 | 基本反映原文信息，遗漏部分次要内容，或有轻微偏差但不影响核心信息 |
| 60-69分 | 反映部分核心信息，有明显遗漏或偏差，需要补充才能准确理解 |
| 0-59分 | 严重遗漏或歪曲原文核心信息，无法准确传达原文意图 |

**关键检查点**：
- 是否遗漏关键数据、事实或重要观点？
- 是否对原文信息进行不当的扩展或曲解？
- 信息引用是否准确？

### 2. 结构完整性（25分）

| 评分 | 标准 |
|-----|------|
| 90-100分 | 结构非常清晰，逻辑连贯，层次分明，有明确的引言、正文、结尾 |
| 80-89分 | 结构清晰，逻辑基本连贯，层次较分明 |
| 70-79分 | 结构基本合理，有清晰的主线，但部分段落逻辑衔接不够自然 |
| 60-69分 | 结构松散，缺乏清晰的主线，部分内容组织混乱 |
| 0-59分 | 结构混乱，无逻辑主线，内容组织杂乱无章 |

**关键检查点**：
- 是否有清晰的开头、主体和结尾？
- 段落之间的逻辑关系是否明确？
- 信息组织是否符合文章类型的特征（如新闻的倒金字塔结构、教程的步骤结构）？

### 3. 语言质量（20分）

| 评分 | 标准 |
|-----|------|
| 90-100分 | 语言流畅自然，用词准确恰当，表达清晰，符合目标文章类型的风格要求 |
| 80-89分 | 语言较流畅，用词基本准确，表达较清晰，符合类型风格 |
| 70-79分 | 语言基本通顺，偶有表达不畅或用词不当，但不影响理解 |
| 60-69分 | 语言不够流畅，存在多处表达问题或用词错误，影响阅读体验 |
| 0-59分 | 语言问题严重，表达晦涩难懂，用词错误频繁 |

**关键检查点**：
- 语句是否通顺流畅？
- 用词是否准确，是否符合语境？
- 语言风格是否符合目标文章类型（如新闻报道的简洁客观、产品介绍的吸引力）？

### 4. 配图契合度（15分）

| 评分 | 标准 |
|-----|------|
| 90-100分 | 所有配图都与文章内容高度相关，图片理解准确，配图位置恰当，图文形成完美配合 |
| 80-89分 | 大部分配图与文章内容相关，图片理解基本准确，配图位置合理，图文配合良好 |
| 70-79分 | 配图基本相关，图片理解有一定偏差，部分配图位置不够合理 |
| 60-69分 | 部分配图与文章内容关联度低，图片理解存在明显错误，配图位置不当 |
| 0-59分 | 配图与文章内容基本无关或严重冲突，图片理解错误严重，配图随意且影响阅读 |

**关键检查点**：
- 是否对图片内容进行了准确的理解和描述？
- 图片与文章内容的主题是否相关？
- 配图位置是否恰当（是否在相关段落附近）？
- 图片引用的描述是否准确反映了图片内容？
- 图文是否形成良好的视觉和逻辑配合？

### 5. 需求匹配度（15分）

| 评分 | 标准 |
|-----|------|
| 90-100分 | 完全满足用户所有明确需求，包括文章类型、目标受众、风格要求等 |
| 80-89分 | 满足用户主要需求，次要需求基本满足 |
| 70-79分 | 基本满足用户的核心需求，部分细节需求未完全满足 |
| 60-69分 | 部分满足用户需求，存在与需求不符的地方 |
| 0-59分 | 严重偏离用户需求，类型、风格等关键要求未满足 |

**关键检查点**：
- 文章类型是否符合用户需求？
- 是否针对目标受众进行表达？
- 是否满足用户的特殊要求（字数、风格、重点等）？

## 总分计算

**总分计算公式**：
```
总分 = 内容准确性得分 × 0.35 + 结构完整性得分 × 0.20 + 语言质量得分 × 0.15 + 配图契合度得分 × 0.15 + 需求匹配度得分 × 0.15
```

**评级标准**：
- 90-100分：优秀（Excellent）
- 80-89分：良好（Good）
- 70-79分：中等（Average）
- 60-69分：及格（Pass）
- 0-59分：不及格（Fail）

## 评分示例

### 示例1：优秀案例（总分92分）

**文章**：技术博客，将分布式系统技术文档转换为面向开发者的博客文章，包含2张架构图

**评分**：
- 内容准确性：95分（完整准确反映原文所有技术要点，无遗漏）
- 结构完整性：90分（结构清晰，有引言-技术概述-核心要点-实践价值-总结）
- 语言质量：92分（语言流畅专业，术语使用恰当，符合技术博客风格）
- 配图契合度：90分（2张架构图都与文章内容高度相关，配图位置恰当，图文配合完美）
- 需求匹配度：90分（完全符合技术博客类型，面向开发者，重点讲解架构思想）

**总分**：95 × 0.35 + 90 × 0.20 + 92 × 0.15 + 90 × 0.15 + 90 × 0.15 = 33.25 + 18 + 13.8 + 13.5 + 13.5 = 92.05分

**评级**：优秀

### 示例2：中等案例（总分73分）

**文章**：产品介绍文案，包含3张产品功能截图

**评分**：
- 内容准确性：80分（准确反映主要功能和优势，遗漏次要使用场景）
- 结构完整性：75分（结构合理，但部分段落衔接不够自然）
- 语言质量：70分（语言基本通顺，部分用词不够精准）
- 配图契合度：68分（产品截图基本相关，但有1张截图位置不当，图片描述不够准确）
- 需求匹配度：70分（基本符合产品介绍类型，但吸引力不够）

**总分**：80 × 0.35 + 75 × 0.20 + 70 × 0.15 + 68 × 0.15 + 70 × 0.15 = 28 + 15 + 10.5 + 10.2 + 10.5 = 74.2分

**评级**：中等

### 示例3：不及格案例（总分58分）

**文章**：新闻报道，包含1张事件现场照片

**评分**：
- 内容准确性：70分（遗漏了关键事件时间和地点）
- 结构完整性：50分（结构混乱，缺乏清晰的导语和结尾）
- 语言质量：55分（语言不通顺，多处表达错误）
- 配图契合度：45分（配图与新闻事件无关，图片理解错误严重，配图位置不当影响阅读）
- 需求匹配度：60分（不符合新闻报道的简洁客观风格）

**总分**：70 × 0.35 + 50 × 0.20 + 55 × 0.15 + 45 × 0.15 + 60 × 0.15 = 24.5 + 10 + 8.25 + 6.75 + 9 = 58.5分

**评级**：不及格

## 改进建议

针对不同评分区间，提供相应的改进建议：

### 90-100分（优秀）
- "文章质量优秀，完全满足需求。如需进一步优化，可考虑..."
- 建议：微调语言风格、补充更多细节等

### 80-89分（良好）
- "文章质量良好，小幅优化可达到优秀水平。建议..."
- 建议：补充遗漏的信息、优化段落衔接、调整用词精准度

### 70-79分（中等）
- "文章基本满足需求，但有一定提升空间。建议重点改进..."
- 建议：补充关键信息、调整文章结构、优化语言表达

### 60-69分（及格）
- "文章勉强及格，存在较多问题。建议..."
- 建议：重新组织结构、补充核心信息、大幅优化语言表达、重新评估配图相关性并调整

### 0-59分（不及格）
- "文章质量不合格，建议重新生成。主要问题：..."
- 建议：明确指出问题并提供重新生成的方向，重点解决配图不相关、图片理解错误等问题
